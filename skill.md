# Skill名

Screenshot Assignment Todo Extractor

# Purpose

スクリーンショット画像から課題、締切、予定を抽出し、時系列順のToDoリストに整理する。学習管理システム、カレンダー、チャット、メール、授業ページなどの画像に含まれる日本語・英語の課題情報を扱う。

# Input

- スクリーンショット画像
- 画像内に含まれる日本語または英語テキスト
- 想定される情報:
  - 課題名、予定名、レポート名、宿題名
  - 締切日、期限、提出期限、deadline、due
  - 時刻

# Process

1. 画像をOCRしやすい形に前処理する。
   - 小さい文字に対応するため画像を拡大する。
   - グレースケール化する。
   - コントラストを上げる。
   - 日本語と英語のOCRを併用する。
2. OCR結果を行単位に正規化する。
   - 全角英数字を半角寄りに正規化する。
   - 余分な空白、罫線、記号を整理する。
   - 改行崩れがある短い行は前後の行と結合候補にする。
3. 日付と時刻を検出する。
   - 対応日付例: `6/20`、`6月20日`、`2026-06-20`、`2026年6月20日`
   - 対応時刻例: `23:59`、`9:00`
   - 年がない場合は現在年を推定年として補う。
4. 日付の前後2〜3行を課題名・予定名候補として評価する。
   - 同じ行に日付以外の文字がある場合は優先候補にする。
   - 前後行に「課題」「宿題」「レポート」「提出」「締切」「期限」「予定」「deadline」「due」「assignment」などがあれば高く評価する。
   - UIノイズや日付・時刻だけの行は候補から除外する。
5. 課題名、締切日、時刻を分離する。
   - 課題名から日付、時刻、締切ラベルを除去する。
   - 日付がない課題候補は締切日を `期限不明` とする。
   - 時刻がない場合は `時刻未検出` とする。
6. ToDoを時系列順に並べる。
   - 日付と時刻がある項目を先に並べる。
   - `期限不明` の項目は末尾に置く。
   - 重複候補は統合する。

# Output

ToDoリスト形式で以下を出力する。

- 課題名または予定名
- 締切日
- 時刻
- チェック可能なToDo項目
- 必要に応じてOCR読み取り結果

# Example

入力OCRテキスト:

```text
第3回 レポート課題
提出期限: 2026年6月20日 23:59
来週の予定
6/24 10:30 ゼミ発表
```

出力ToDo:

```text
[ ] 第3回 レポート課題
    締切日: 2026-06-20 / 時刻: 23:59

[ ] ゼミ発表
    締切日: 2026-06-24 (年推定) / 時刻: 10:30
```
